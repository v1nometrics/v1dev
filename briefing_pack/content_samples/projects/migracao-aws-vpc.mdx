## Objetivo
Consolidar aplicações em uma VPC própria, reduzindo superfície de ataque e melhorando latência EC2↔RDS.

## Componentes
- VPC + subnets privadas
- SGs restritivos
- RDS Postgres
- EC2 ARM64 + Docker
- ECR para distribuição de imagens
- CloudWatch logs/alarms

## Runbook (resumo)
1) build → push ECR
2) pull na EC2
3) migrate DB
4) healthcheck
5) rollback via tag anterior
